# PRD｜人生副本视频工作站（Polaris Story Studio）

> 文档格式：PRD-HRM（Hierarchical Requirement Model，分层需求模型）
> 版本：v0.1（规划稿）  归属：`d:\polaris\polaris-app`  日期：2026-06-13
> 一句话：把豆包截图里那套「AI 人生模拟器短视频」5 步工作流，固化成 Polaris 桌面端可一键跑通的三模块产品。

---

## 0. 文档怎么读（HRM 分层约定）

本 PRD 用三层结构，每条需求都能定位到代码：

| 层 | 代号 | 含义 | 示例 |
|---|---|---|---|
| L1 目标层 | `G-x` | 业务目标 / 北极星 | G-1 一键产出人生副本视频 |
| L2 能力层 | `C-x.y` | 模块能力（对应一个 Vue 组件 / store） | C-2.1 6 级剧本生成 |
| L3 任务层 | `T-x.y.z` | 可落地的开发任务（对应函数 / 文件改动） | T-2.1.1 节奏模板注入 prompt |

每条 L3 都带 **验收标准（AC）** 与 **落点文件**，开发时一条一条勾。

## 1. 背景与参考（L1）

### 1.1 灵感来源
参考素材是一段「AI 人生模拟器视频生成」工作流（豆包对话截图），核心主张：
**选一个有冲突感的人设（偷车贼 / 杀猪匠 / 单身一生）→ 第一人称 6 级分层剧本 → 低沉叙事 TTS + 环境音 → AI 配画面 → 竖版 9:16 发布**。

### 1.2 直接可借鉴的开源项目（写进「设置 → 参考库」只读卡片）
| 项目 | 用途 | 我们抄什么 |
|---|---|---|
| **LifeReloaded**（github.com/hamutama/LifeReloaded） | GPT 驱动人生重开模拟器，含完整提示词 | 抄它的「人生阶段递进 + 关键抉择」提示词结构，改造成 6 级剧本模板 |
| **lifeRestart**（github.com/VickScarlet/lifeRestart） | 经典人生重开模拟器，可导出事件流 | 抄它的「随机事件 / 属性」表，做节奏模式里的「转折事件库」 |
| **Parallel Lives**（pingcap.com/blog/build-ai-powered-life-simulator） | AI 平行人生模拟器，分支剧情 | 抄它的「分支多结局」思路，做「多集系列」扩展位 |

> 说明：这些项目仅作**提示词与剧情结构参考**，不引入其前端代码、不引入运行时依赖（遵循「怎么小怎么来」原则）。

### 1.3 现状盘点（已有，不重造）
polaris-app 已经存在的可复用资产：

| 现有资产 | 文件 | 在本产品里的角色 |
|---|---|---|

| 视频课程工作台 | `src/components/VideoCourseStudio.vue`（1538 行，已含分镜 / 生图 / 15 套美术风格） | **模块三：视频**（改造为人生副本工作流） |
| 美术风格库 | `lib/storyStyles.ts`（15 套画风，拼进生图 prompt） | 视频模块「画风选择」直接用 |

**结论：地基齐全。本 PRD 的工作 = 裁剪导航 + 在视频模块注入「人生副本 5 步工作流」+ 在设置补齐声图 / 语音供应商。**

## 2. 产品目标（L1）

| 代号 | 目标 | 衡量指标 |
|---|---|---|
| **G-1** | 一条龙跑通：输入人设 → 输出竖版叙事视频草稿 | 从「生成剧本」到「导出分镜+配音+画面」≤ 10 分钟，零手写代码 |
| **G-2** | 节奏可套用：剧情按可选「节奏模式」自动分层 | ≥ 4 套节奏模板，切换后剧本结构实时变化 |
| **G-3** | 三模块极简：只保留 对话 / PPT / 视频 | 左栏只剩 3 个一级入口 + 1 个设置 |
| **G-4** | 高级感界面：墨蓝水墨主视觉，无 emoji | 通过 design_wukong_style 验收（主色 #2c4661） |
| **G-5** | 模型自由接入：LLM / 声图 / 语音都可在设置里配 | 每类 ≥ 2 个内置预设 + 自定义 OpenAI 兼容端点 |

## 4. 视频模块详细需求（L2 → L3）

### C-4.1 步骤一：人设选题
| 任务 | AC（验收标准） | 落点 |
|---|---|---|
| **T-4.1.1** 人设卡输入：职业/人生主题 + 一句话冲突点 | 必填「主题」，自动生成标题「今天体验 XX 的一生」 | 视频模块新增 `PersonaCard` 区块 |
| **T-4.1.2** 灵感库：内置 ≥ 12 个高冲突人设（偷车贼/杀猪匠/单身一生/外卖骑手/守墓人/赌徒…） | 点一下填入卡片 | 新增 `lib/lifePersonas.ts` |
| **T-4.1.3** 「职业点」网状图（可选增强） | 给一张可点的人生节点网站图，点节点 = 选人设 | 复用 `cytoscape`（已装），新增轻量节点图 |

### C-4.2 步骤二：生成剧本（节奏引擎，核心）
这是「左栏第一个选项就能生成剧本」的落点。

| 任务 | AC | 落点 |
|---|---|---|
| **T-4.2.1** 6 级分层剧本生成：第一人称，每级 100–150 字，含场景/动作/心理 | LLM 返回结构化 `levels[6]`，每级有 `scene/action/psych/caption` 字段 | 新增 `lib/lifeScript.ts` + 调 providers LLM |
| **T-4.2.2** **节奏模式库**（≥ 4 套，可套用） | 切换节奏，剧本的级数/张力曲线/转折位实时变化 | 新增 `lib/rhythmModes.ts`（见 §4.2.1） |
| **T-4.2.3** 提示词模板：把截图那段「写 XX 的一生 6 级剧情…」固化为可编辑模板 | 模板可见、可改、可重置默认 | `lifeScript.ts` 内置 `PROMPT_TEMPLATE` |
| **T-4.2.4** 剧本可编辑：生成后逐级手改，改完锁定进下一步 | 编辑后状态持久化到草稿 | 视频模块 `ScriptEditor` 区块 |

#### 4.2.1 节奏模式库（T-4.2.2 展开）— 套各种节奏的核心
每套节奏 = 一条**张力曲线** + **级数** + **转折点位置** + **结尾处理**：

| 节奏模式 | 级数 | 张力曲线 | 适用 | 说明 |
|---|---|---|---|---|
| **平步沉沦**（默认） | 6 | 缓起 → 逐级加压 → 第5级崩塌 → 第6级余味 | 偷车贼/赌徒类「下坠人生」 | 截图原版节奏 |
| **高开低走** | 6 | 第1级即巅峰 → 持续滑落 → 结尾顿悟 | 单身一生/中年危机 | 反差感强 |
| **谷底反弹** | 7 | 低开 → 第4级触底 → 后段逆袭 | 励志/翻身题材 | 适合正能量 |
| **平行抉择** | 5+分支 | 第3级出现岔路 → 双结局 | 平行人生/多集系列 | 借鉴 Parallel Lives |
| **快剪高燃** | 4 | 短促、每级强钩子 | 卡点快节奏 | 每级≤80字 |

> 节奏数据结构（写进 `rhythmModes.ts`）：
> ```ts
> interface RhythmMode {
>   id: string; name: string; levels: number;
>   curve: number[];            // 每级张力 0-100，驱动配乐/画面冷暖
>   twistAt: number;            // 转折级序号
>   branchAt?: number;          // 分支级（可选）
>   wordPerLevel: [number,number]; // 字数区间
>   promptHint: string;         // 拼进 LLM 的节奏指令
> }
> ```

### C-4.3 步骤三：叙事配音（TTS）
| 任务 | AC | 落点 |
|---|---|---|
| **T-4.3.1** 逐级生成低沉叙事音 | 每级一段音频，可试听、可重生 | 复用 `VoiceSettings` 接的 TTS 供应商 |
| **T-4.3.2** 叙事音色预设：低沉男声/沧桑/旁白感（模仿故事叙述级有感觉的语音） | ≥ 3 个内置音色，按节奏张力推荐 | `lib/narratorVoices.ts` |
| **T-4.3.3** 环境音叠加（雨声/键盘声/低频氛围） | 可选叠加，音量可调 | 本地静音素材 or TTS SSML |

### C-4.4 步骤四：画面生成（声图）
| 任务 | AC | 落点 |
|---|---|---|
| **T-4.4.1** 逐级配图：每级剧本 → 1 张竖版图 | 调声图模型，画风统一（拼 `storyStyles` 后缀） | 复用 VideoCourseStudio 现有生图链路 |
| **T-4.4.2** 画风选择：15 套画风（已有） + AI 自由发挥 | 切画风全片重绘 | `lib/storyStyles.ts`（现成） |
| **T-4.4.3** 张力驱动构图：高张力级自动偏暗/特写 | 张力值映射到 prompt 修饰 | 用 §4.2.1 的 `curve` |

### C-4.5 步骤五：合成导出
| 任务 | AC | 落点 |
|---|---|---|
| **T-4.5.1** 时间轴预览：分镜 = 图 + 字幕 + 配音 + LEVEL 标记 | 竖版 9:16 预览可播放 | 视频模块 `Timeline` 区块 |
| **T-4.5.2** 导出包：图片序列 + 音频 + 字幕(srt) + 剪映可读清单 | 一键导出到工作目录 | 复用 Tauri `plugin-fs` |
| **T-4.5.3** 发布文案模板：固定话术 + #人生副本 #体验100种人生 | 一键复制 | `lib/publishTemplate.ts` |

## 6. UI / 视觉规范（L2，G-4）

- **主色**：墨蓝 `#2c4661`；选中态淡灰 `#ebebe8`；朱砂仅用于警告（遵循 design_wukong_style）。
- **绝不用 emoji**，图标用 `@lucide/vue`（已装）+ 自绘水墨线条。
- **步骤条**：横向 5 段，已完成段填墨蓝，当前段水墨描边脉冲。
- **节奏曲线**：用一条手绘感折线可视化 `curve[]`，张力高处偏朱砂、低处偏墨蓝。
- **整体气质**：留白、克制、电影感深色预览区 + 浅色操作区对比。

## 7. 数据流（端到端，一图）

```
人设卡(persona)
   │  + RhythmMode
   ▼
LLM(lifeScript) ──► levels[6]{scene,action,psych,caption,tension}
   │                      │                    │
   ├──► TTS(narrator) ──► audio[level]         │
   ├──► 声图(style+tension) ──► image[level]    │
   ▼                                            ▼
Timeline 合成 ──► 导出包(imgs+audio+srt+清单) + 发布文案
```

草稿模型 `StoryProject` 持久化到工作目录（JSON），随时续编。

## 9. 范围与非目标

**做**：三模块、人生副本 5 步、节奏库、模型接入、导出剪映可读包。
**不做（本期）**：真·视频渲染合成（交给剪映/Runway，我们只出料）、账号体系、云端协作、多集系列自动化（仅留 `branchAt` 扩展位）。
