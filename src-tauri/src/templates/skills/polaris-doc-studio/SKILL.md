# Polaris 文档工坊（Word 教案）

> 输入课题/教材片段/讲稿 → 你把它编排成 **spec**（`polaris.doc.json`）→ 引擎转成**原生可编辑**的 `.docx`。
> **你的职责是内容与结构决策，不是排版。** 字体、字号、行距、页边距、表格线、编号由引擎按主题确定性渲染，你不用管也管不了。

## 这个技能的核心约定（先读这段）

**Word 教案 = spec 路线，这是唯一主路径。**你**不写 HTML、不调 python-docx、不截图**，只产出一个 JSON：

```
polaris.doc.json  →  polaris-forge spec-docx  →  教案.docx
```

为什么这么设计：引擎手写 OOXML，产物是真段落/真表格/真公式域，教师拿到 `.docx` 能直接改字、插行、打印。且**预览即导出**——桌面端预览用的是同一份 spec 同一套主题表，预览一个样、导出另一个样是不可接受的回归。

**铁律**：spec 是唯一真源。预览、导出、继续改稿全部基于它。改稿就改 spec 再重转，**绝不**另起新文件。

## 一、spec v1 格式（权威，与引擎逐字对齐）

```json
{
  "version": 1,
  "theme": "qingjiao",
  "page": { "size": "a4", "mt": 25.4, "mr": 31.7, "mb": 25.4, "ml": 31.7,
            "header": "", "footer": "第 {page} 页" },
  "blocks": [
    {"type": "title", "text": "导数在函数单调性、极值与最值中的应用"},
    {"type": "subtitle", "text": "——高考一轮复习专题突破（第1课时）"}
  ]
}
```

顶层四个键：

- `version` 恒为 `1`。
- `theme` 取下面 5 套主题之一（缺省 / 写错 → 回退 `qingjiao`）。
- `page` **全字段可省**。边距单位是**毫米**；`size` 取 `a4`(默认) 或 `letter`；`footer` 里的 `{page}` 导出时换成 Word 真页码域（`header`/`footer` 不写就没有页眉页脚）。
- `blocks` 是块数组，**顺序即文档顺序**，不能为空。

### 5 套主题（`theme` 取值）

| id | 名称 | 正文 / 标题字体 | accent | 正文磅值 | 行距 | 适用 |
|---|---|---|---|---|---|---|
| `qingjiao` | 青教赛范式 | 微软雅黑 / 微软雅黑 | `#2C4661` | 12 | 1.6 | **默认**。教案、教学设计、说课稿、公开课材料，通吃 |
| `songti` | 公文宋体 | 宋体 / 黑体 | `#8C2B2B` | 12 | 1.75 | 通知、总结、课题申报、上报材料 |
| `kaiti` | 楷体清雅 | 楷体 / 微软雅黑 | `#3E6B4F` | 12 | 1.7 | 语文/历史/德育类，有文气 |
| `modern` | 现代蓝 | 微软雅黑 / 微软雅黑 | `#2563EB` | 11.5 | 1.65 | 导学案、学案、信息技术类 |
| `warm` | 暖橘手账 | 微软雅黑 / 微软雅黑 | `#C2643A` | 12 | 1.7 | 小学段、班会课、家长会材料 |

**大小写敏感，照抄 id。** 教案一律用 `qingjiao`，除非用户明确要别的气质。

### 14 种块类型（`blocks[].type`）

**没有嵌套**——表格单元格里也只放纯文本 + 行内标记。未列出的字段引擎不读，写了白写。

| type | 语义 | 用到的字段 |
|---|---|---|
| `title` | 文档大标题（自动居中、主题标题字号） | `text` |
| `subtitle` | 副标题（自动居中、楷体灰） | `text` |
| `h1` | 一级标题，渲染时**自动加主题色方块前缀**——`text` 里**绝不要再写「■」** | `text` |
| `h2` / `h3` | 二三级标题 | `text` |
| `p` | 正文段，**默认首行缩进 2 字符** | `text`, `indent` |
| `bullet` | 无序要点，渲染器自动出「·」——`text` 里**绝不要再写「· 」「- 」「• 」** | `text` |
| `num` | 有序要点，**序号引擎自动排**——`text` 里**绝不要写「1.」「（1）」** | `text` |
| `quote` | 引文/寄语（左侧主题色竖条 + 楷体） | `text` |
| `callout` | 提示框（浅底描边），用于重点/难点/易错/学法提示 | `head`, `text` |
| `table` | 表格 | `rows`, `head0`, `widths` |
| `image` | 插图 | `src`, `w`, `cap` |
| `hr` | 分隔线 | — |
| `pagebreak` | 硬分页 | — |

**通用可选字段（任何文字块都能加）**：

- `align`：`left`(默认) / `center` / `right` / `both`(两端对齐)
- `indent`：`first`(`p` 的默认) / `none`（取消首行缩进）
- `pad`：整块左缩进，单位 em（`bullet` 默认 1）
- `size`：覆盖主题字号，单位**磅(pt)**
- `color`：主题色词 `ink` / `accent` / `muted` / `line` / `soft`，或 `#RRGGBB`
- `bold` / `italic`：布尔

**`table` 专属**：

- `rows`：`string[][]`，每格是含行内标记的字符串；**行长不齐按最长行补空**，但请自己对齐，别指望补齐后好看
- `head0`：首行是否表头（默认 `true`，表头自动加底色 + 居中 + 加粗）。**板书设计那种单格表要写 `"head0": false`**
- `widths`：各列宽比例数组，**长度必须等于列数**，内部归一化；缺省等分

**`image` 专属**：

- `src`：本地**绝对路径**（png/jpg），图必须先真存在于磁盘
- `w`：宽度占正文宽的百分比 1–100（默认 100）
- `cap`：图注

### 6 种行内标记（所有文字字段通用）

**故意不做富文本树**（点字直改要求叶子是纯字符串）。所有 `text` / `head` / 表格单元格都支持同一套轻量语法：

| 写法 | 效果 |
|---|---|
| `**粗**` | 加粗 |
| `*斜*` | 斜体 |
| `__下划线__` | 下划线 |
| `~~删除~~` | 删除线 |
| `` `代码` `` | 等宽 |
| `$x^2+1$` | **行内公式**（导出成真 OMML，Word 里可点开编辑） |

解析顺序（双端一致）：先切出 `` `code` `` 与 `$math$`（内部不再解析），再依次 `**` → `__` → `~~` → `*`。

**公式子集**（导出 LaTeX → OMML 只覆盖这些，超出的降级成纯文本）：
`\frac{}{}` · `^{}` · `_{}` · `\sqrt{}` / `\sqrt[n]{}` · `\left(...\right)` · `\sum` / `\int`（带上下限） · `\sin` 等函数名 · `\vec{}` / `\bar{}` · 裸符号。

> **理科教案的公式一律走 `$...$`**，不要用 `x^2`、`f'(x)>0` 这类裸文本 —— 那在 Word 里就是丑陋的普通字符，评委一眼看出不专业。
> 正确：`$f'(x)>0$`、`$f(x)=x^3-3x$`、`$\frac{1}{2}x^2-a\ln x$`、`$\sqrt{a^2+b^2}$`。

### 完整可复制的最小样例

```json
{

  "theme": "qingjiao",
  "page": { "size": "a4", "footer": "第 {page} 页" },
  "blocks": [
    {"type": "title", "text": "带电粒子在匀强磁场中的运动"},
    {"type": "subtitle", "text": "——高考磁场专题突破（第1课时）"},
    {"type": "table", "head0": false, "widths": [1, 1.6, 1, 1.6], "rows": [
      ["学　　科", "物理", "课　　型", "高考二轮·专题课"],
      ["授课年级", "高三", "课　　时", "1 课时（45 分钟）"],
      ["教材版本", "人教版·选择性必修第二册", "适用模块", "磁场与洛伦兹力"],
      ["设计者", "＿＿＿", "核心素养", "物理观念·科学思维·科学探究·科学态度"]
    ]},
    {"type": "h1", "text": "一、课标要求与考情分析"},
    {"type": "p", "text": "《课标》要求：认识洛伦兹力，会判断其方向，能分析带电粒子在匀强磁场中的圆周运动。"},
    {"type": "p", "text": "（近五年高考考情）", "indent": "none"},
    {"type": "bullet", "text": "考查频率：全国卷每年必考，选择题或压轴计算题，分值 6～16 分。"},
    {"type": "bullet", "text": "失分点：圆心定位不准；半径公式 $r=\\frac{mv}{qB}$ 与周期 $T=\\frac{2\\pi m}{qB}$ 混淆。"},
    {"type": "callout", "head": "难点", "text": "圆心与半径的几何定位——「速度垂线交点即圆心」。"},
    {"type": "h1", "text": "六、教学过程"},
    {"type": "table", "widths": [1.1, 3, 2.2, 2.2], "rows": [
      ["教学环节", "教师活动", "学生活动", "设计意图"],
      ["环节一\n情境导入\n(5′)", "播放回旋加速器动画，追问：粒子为什么走圆？", "回答洛伦兹力始终垂直于速度、只改变方向。", "由真实装置切入，把「向心力来源」这一旧知激活，为后续定量推导铺路。"]
    ]}
  ]

```

> 注意 JSON 里反斜杠要转义：LaTeX 的 `\frac` 在 JSON 字符串里写成 `\\frac`。
> 表格单元格里的 `\n` 是真换行（JSON 写 `\n`），用来做「环节一 / 情境导入 / (5′)」这种三行式环节名。

## 二、青教赛教案范式（**本技能的灵魂，默认骨架**）

用户说「写教案」「教学设计」「说课稿」「公开课材料」时，**除非明确另有要求，一律按这十节骨架产出**。这是青年教师教学基本功大赛（青教赛）与各级公开课评审通用的范式，评委按节找分。

### 骨架总览

```
title      课题名（不带「教案」二字，写真实课题）
subtitle   ——学段/专题定位（第N课时）
table      基本信息表（4 行 × 4 列，head0:false）
h1 一、课标要求与考情分析
h1 二、学情分析
h1 三、教学目标         ← 三维目标（或学科核心素养分点）
h1 四、教学重点与难点
h1 五、教学方法与手段
h1 六、教学过程         ← 四栏表：教学环节·教师活动·学生活动·设计意图
h1 七、板书设计         ← 单格表格（head0:false）
h1 八、分层作业设计
h1 九、教学反思（预设）
h1 十、课程思政融入
```

**h1 的 `text` 就写「一、课标要求与考情分析」——序号是内容的一部分（这是教案惯例，不是自动编号），但「■」方块由渲染器加，你不要写。**

### 逐节写法与颗粒度

#### 标题与副标题
- `title`：真实课题名，不含「教案」「教学设计」字样。例：`导数在函数单调性、极值与最值中的应用`。
- `subtitle`：`——` 开头的定位副标题，含课时序号。例：`——高考一轮复习专题突破（第1课时）`。

#### 基本信息表（4×4，`head0: false`）
标签列窄、内容列宽，`widths` 用 `[1, 1.6, 1, 1.6]`。标签用**全角空格撑成两字宽**（`学　　科` / `课　　型` / `课　　时`），这是排版惯例。

八对标准字段：学科·课型 / 授课年级·课时 / 教材版本·适用模块 / 设计者·核心素养。
- 「课型」写具体：`高考一轮复习·专题课` / `新授课` / `实验探究课` / `复习巩固课` / `讲评课`。
- 「课时」写 `1 课时（45 分钟）`。
- 「设计者」留 `＿＿＿`（全角下划线），让教师自己填。
- 「核心素养」写**本学科的**素养词，用 `·` 连接：数学=数学抽象·逻辑推理·数学运算·直观想象；语文=语言建构与运用·审美鉴赏与创造·思维发展与提升·文化传承与理解；物理=物理观念·科学思维·科学探究·科学态度与责任；英语=语言能力·文化意识·思维品质·学习能力；历史=时空观念·史料实证·历史解释·家国情怀；地理=区域认知·综合思维·地理实践力·人地协调观；政治=政治认同·科学精神·法治意识·公共参与。

#### 一、课标要求与考情分析
- 一个 `p`：**引原文**。写清课标名称（如《普通高中数学课程标准（2017年版2020年修订）》）+ 该课题对应的具体要求句。不要写「课标要求学生掌握本节内容」这种废话。
- 一个 `p`（`indent: "none"`）小标：`（近五年高考考情）` 或 `（教材地位与考情）`。
- 3 条 `bullet`：**考查频率**（分值、题型、出现在第几题）、**命题趋势**（从什么演进到什么）、**失分点**（具体到操作错误，不是「不认真」）。

#### 二、学情分析
一个 `p`（可两段）。必须写成「**已有什么 → 缺什么 → 因此本课定位在哪**」的三段式，**不要写「学生基础参差不齐、学习积极性有待提高」这种放之四海皆准的空话**。
范例口吻：「学生在必修阶段已掌握……；但在……时缺乏……，常出现……。本节课定位于把零散方法结构化为……，并突破……难点。」

#### 三、教学目标
三个 `p`（`indent: "none"`）做小标 + 每标下 1–2 条 `bullet`：
- `（1. 知识与技能）` —— 或按学科素养命名，如 `（1. 语言建构与运用）`
- `（2. 过程与方法（核心素养落点））`
- `（3. 情感态度与价值观）`

每条 bullet 用**可观测行为动词**开头：说出 / 判定 / 熟练由…判定 / 掌握…流程 / 能识别 / 会用…求解。**禁止「理解」「体会」单独成目标**（不可测）——要写成「通过……体会……」。

#### 四、教学重点与难点
两个 `p`（`indent: "none"`），或用两个 `callout`：
- `【重点】` + 一句话，指向**可迁移的方法/流程**。
- `【难点】` + 一句话，**必须说明难在哪**（`含参数函数单调性的分类讨论——分类标准的确定与"不重不漏"`），不能只重复重点。

#### 五、教学方法与手段
3 条 `bullet`：
- `教法：` 问题导学 + 变式训练 + 错例辨析；以「一题一议、一议一得」驱动。
- `学法：` 独立求解—小组互评—板演纠错，经历「做—比—改」的循环。
- `手段：` 具体到工具（几何画板动态演示 / 实物投影展示学生答卷 / 配乐诵读 + 意境图投影）。

#### 六、教学过程（**全篇的重头，占一半篇幅**）
一个 `table`，`widths` 用 `[1.1, 3, 2.2, 2.2]`，表头行 `["教学环节", "教师活动", "学生活动", "设计意图"]`。

**5 个环节**是标准配置（时长之和 = 45′），环节名用 `\n` 分三行：序号行 / 名称行 / 时长行，如 `环节三\n难点突破\n含参讨论\n(18′)`。典型节奏：

| 环节 | 名称 | 时长 |
|---|---|---|
| 环节一 | 情境导入 / 知识唤醒 | 5–6′ |
| 环节二 | 建模示范 / 典例精讲 | 10–12′ |
| 环节三 | 难点突破（本课核心） | 15–18′ |
| 环节四 | 变式提升 / 迁移训练 | 8–12′ |
| 环节五 | 小结建构 | 4–5′ |

- **教师活动**：写**具体动作 + 真题干 + 设问链**。要出现真的例题、真的追问。例：`例2 讨论 $f(x)=\frac{1}{2}x^2-a\ln x\ (x>0)$ 的单调性。设问链：①定义域？②$f'(x)=?$ ③按 $a\le 0$ 与 $a>0$ 分类。` **不要写「讲解例题」「引导学生思考」。**
- **学生活动**：写学生**做什么、产出什么**（口答/板演/小组讨论确定分类标准/互评是否不重不漏）。不要写「认真听讲」。
- **设计意图**：**这一栏是评委的重点，必须写实**。要回答「这样安排解决了什么教学问题」，与前面的学情/难点呼应。至少 20 字，句式是「以……（手段），达成……（认知目的），为……铺垫 / 纠正……的常见误区」。
  - ✅ `以简单题激活已有经验，提炼可迁移的解题程序，为难点分散铺垫。`
  - ✅ `本课核心。以"分界点来自 $f'(x)=0$ 的根是否落在定义域内"统一分类逻辑，攻克难点。`
  - ❌ `激发学生兴趣，提高课堂效率。`（空话，一律不许）
  - ❌ `巩固所学知识。`（谁都能写，等于没写）

#### 七、板书设计
一个**单格表格**：`{"type": "table", "head0": false, "rows": [["…"]]}`，格内用 `\n` 分行，模拟真黑板版面：第一行居中写课题，下面 3–4 条编号主干（一、二、三、四），用 `→` `｜` `⇔` 等符号表现结构关系。**这是「板书」不是「提纲」——要能照着抄到黑板上。**

#### 八、分层作业设计
3 条 `bullet`，**三档必须真有梯度**（不是同一题换数字）：
- `【基础·必做】` 直接套本课流程的常规题。
- `【提升·必做】` 含参 / 变式 / 需要选择方法的题。
- `【挑战·选做】` 逆向、开放、或指向下节课的延伸（`并思考"极值点偏移"问题的求解思路`）。

#### 九、教学反思（预设）
一个 `p`。写**授课前的预判**，三件事：本设计的优势在哪、实施时最可能出的问题（时间不够 / 教师包办讨论 / 学生板演拖堂）、**给不同班情的调整预案**（`对基础薄弱班，可将例2参数范围简化为 $a>0$ 单一情形，再逐步放开`）。

#### 十、课程思政融入
一个 `p`，2–3 句。**必须从本课的知识内容里自然长出来**，不能贴标签：
- ✅ 数学：`借"分类讨论必须不重不漏"渗透"考虑问题要全面、辩证"的思维品格。`
- ✅ 语文：`在品鉴古典诗词的炼字之美中，厚植文化自信，涵养高雅审美情趣，实现"以美育人、以文化人"。`
- ❌ `培养学生的爱国主义精神和社会主义核心价值观。`（与本课无关，套话）

### 范式变体

- **说课稿**：把「六、教学过程」换成「教学过程与设计说明」，每环节后补一句「之所以这样设计是因为……」；加一节「教材分析」放在学情之前。
- **导学案**（发给学生的）：删掉教师活动/设计意图/教学反思/课程思政，改成「学习目标 → 预习自测 → 课堂探究（留空白填写区）→ 当堂检测 → 我的疑问」，主题可换 `modern`。
- **多课时**：一个 spec 一课时。多课时用 `pagebreak` 分隔，每课时重起 `title`+`subtitle`。

## 三、制作流程

### 1. 读懂输入
用户可能给：课题名、教材页码/截图、一份讲稿、素材文件绝对路径（**先 Read**）。
**信息不足时不要停下来问一长串问题**——按最常见情形做合理默认（学段从课题推断、课时按 1 课时 45 分钟、教材版本用该学科主流版本并在基本信息表里让教师可改），把完整教案先交出来，末尾用一两句说明「哪些字段按默认填写，可直接告诉我改」。

### 2. 编排 spec（先想结构再写字）
按上面十节骨架列出块序列，然后逐节填内容。**教学过程表要先想清 5 个环节的认知递进链**，再写教师/学生活动。

### 3. 存到产物目录
文件名**必须**是 `polaris.doc.json`，存到本次会话的产物目录。
（前端靠这个名字找它做预览和兜底转换，**改名整条链路瘫痪**。）

**长教案要分次写盘**：先存一份只含前几节的**合法** JSON（`blocks` 数组必须闭合），再增补到全量——桌面端预览是逐块点亮的，每保存一次用户就多看到一段，不用对着空屏干等。

### 4. 转 .docx
```bash
polaris-forge spec-docx --spec=<产物目录>/polaris.doc.json --out=<产物目录>/教案.docx
```
CLI 在 `~/Polaris/bin/`（Windows 为 `polaris-forge.exe`）。`--out` 的文件名用课题名更友好，如 `导数与函数单调性·教学设计.docx`。

**CLI 不存在也不用慌**：把 spec 按 `polaris.doc.json` 存好即可，Polaris 桌面端会自动调内置引擎完成转换。**不要**因为 CLI 缺失就改去写 HTML / Markdown / 调 python-docx —— 那会毁掉「预览即导出」与可编辑性。

### 5. 回答末尾用**绝对路径**列出产物文件
两个都要列：`polaris.doc.json`（源稿，改稿改它）和 `.docx`（成品）。

## 四、内容硬规矩（**先过这几条，再谈文采**）

### ① 不要手写渲染器负责的符号
`h1` 里写「■」、`bullet` 里写「· 」「- 」「• 」、`num` 里写「1.」「（1）」—— **一律双份**，导出后是「■ ■ 一、…」「· · 考查频率」。渲染器已经加了，你只写文字。

### ② 「设计意图」不许空话
教学过程表每一行的第四栏必须写实、必须与该行的教师/学生活动一一对应。凡是换个课题也照样成立的句子（「激发兴趣」「巩固知识」「提高效率」「培养能力」），一律重写。**这是整份教案里评委看得最细的一栏。**

### ③ 公式一律 `$...$`
理科教案出现的每一个数学表达式、化学式上下标、物理公式都走行内公式。裸写 `x^2`、`f'(x)>0`、`H2O` 是不合格的。

### ④ 表格别越界
- 基本信息表 4 列、教学过程表 4 列，`widths` 长度必须等于列数。
- 单元格里只放纯文本 + 行内标记 + `\n` 换行，**不能嵌套块**。
- 教学过程表单格文字别超 120 字，超了就拆环节——Word 里一行撑满一页很难看。

### ⑤ 篇幅与密度
- 一课时教案总长 **1200–2200 字**为宜（不含表格线）。低于 800 字是敷衍，超过 3000 字打印超 4 页、评委看不完。
- 教学过程表占全篇约一半篇幅，其余九节各 1 段到 3 条要点。
- **每节都要有实质内容**，宁可短，不许留「（略）」「待补充」。

### ⑥ 学科口吻
数学/物理：程序化、步骤化，强调「流程 + 分类标准 + 易错」。
语文/历史/政治：强调文本细读、情境、价值引领，多用「涵泳」「情境」「素养」而非「刷题」。
英语：教学过程里的例句/题干用英文原文，讲解用中文。
不要用同一套模板套所有学科。

### ⑦ 不要在 spec 里塞 Markdown 结构
`# 标题`、`- 列表`、`1. 编号`、`> 引用`、`| 表格 |` 会被原样当正文渲染出来。结构全部用块类型表达：标题用 `h1/h2`、列表用 `bullet/num`、引用用 `quote`、表格用 `table`。**只有行内的 `**` `*` `__` `~~` `` ` `` `$` 这六种标记会被解析。**

## 五、改稿协议

用户说「第三节学情写细一点」「把教学过程改成 6 个环节」「换成宋体公文风」「加一节教材分析」时：

1. **直接改原 `polaris.doc.json`**，文件名不变，**别另起新文件**（写死新文件名 = 用户认识的那份纹丝不动，看起来就是没保存）。
2. 重新跑 `polaris-forge spec-docx` **覆盖导出同名 `.docx`**——覆盖上次那一份，不要生成 `教案_v2.docx`。找不到上次的 `.docx` 才用兜底名 `教案.docx`。
3. CLI 不可用则改完 spec 即可（桌面端按 mtime 判旧，会自动重转）。

只改 spec 不重转 docx 是最常见的疏忽——用户拿到的导出会永远停在第一版。**能跑 CLI 就一定重跑。**

改稿时**只动要改的块**，其余块原样保留（包括用户可能已经手动改过的措辞）。整份重写会把用户的修改冲掉。
