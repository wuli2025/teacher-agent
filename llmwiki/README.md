# RAG × LLM Wiki 知识库

> 一座关于「**怎么用 RAG 增强 llmwiki、怎么让模型自主学习、怎么补全知识**」的方法论知识库。
> 它本身就是一篇 [[LLM-Wiki维护方法论|Karpathy 式 LLM Wiki]]：结构化、双链互联、可被人和模型共同漫游与维护。

---

## 这个库是干什么的

你已经有一套走 **Karpathy「结构化 wiki + 长上下文 + 双链漫游」** 路线的 llmwiki（已具备 LanceDB 向量检索、知识图谱、社区检测、联网搜索）。这个库不替你写代码，而是替你把**该学的、能选的、能落地的 RAG 方法**系统地整理成一张能漫游的知识网，方便你：

1. **系统学习** —— 从基础检索到前沿架构，有路线、有深浅、有取舍。
2. **技术选型** —— 每个技术都写清「是什么 / 怎么干 / 何时用 / 跟 llmwiki 怎么配 / 效果如何」。
3. **指导 llmwiki 演进** —— 把「RAG 当侦察兵、功夫下在结构上」这条主线，落成可执行的配方。

## 一句话主线（来自你的两篇「请教毛主席」）

> **RAG 死的不是它本身，死的是「让它当家做主、撕碎书本盲摸纸片喂给模型」那个用法。**
> 让 RAG 当**侦察兵**（粗筛整篇、三路混合、重排、查询改写、多轮自取），别让它当**主帅**。
> 把劲使在**「结构」和「会取知识」**两门真功夫上 —— 模型越强，这两样越值钱。
