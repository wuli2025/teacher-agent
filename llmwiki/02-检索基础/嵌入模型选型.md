# 嵌入模型选型

> 中文技术 KB 的关键选择。结论先行：**自托管中文首选 BGE-M3 或 Qwen3-Embedding**。

## 怎么评：MTEB / C-MTEB

- **MTEB**（Massive Text Embedding Benchmark）是标准，**MTEB v2 / MMTEB** 为多语版。
- 中文用 **C-MTEB / CMTEB**。⚠️ 警惕榜单过拟合——务必在**自己领域集**上复测。

## 闭源/API 领先（2026 中）

- **OpenAI text-embedding-3-large**：稳的顶级默认，3072 维（Matryoshka 可截断），8191 token。
- **Cohere Embed v4**：多模态（文+图）、**128K token 超长上下文**、Matryoshka、多语强。
- **Voyage voyage-3-large / 3.5**：检索向 MTEB 领先，32K 上下文，int8/binary 量化。
- **Google Gemini Embedding**：2025–26 抬高了上限。

## 开源权重领先（适合自托管/数据不外流）

- **Qwen3-Embedding**（阿里，0.6B/4B/8B）：**开源权重 MTEB v2 榜首**，**中文极强**（8B 在 MTEB 多语 ~70.58）。指令感知、Matryoshka。
- **BGE-M3**（BAAI）：M3 = 多语(100+)、多粒度(8192 token)、**多功能**——一个模型同出 **dense + sparse(字面) + ColBERT 多向量**。MIRACL/MKQA SOTA，MIT 许可可商用。**中文 hybrid KB 的最佳单选**（语义+字面+后期交互一站式）。
- 其他：E5 / multilingual-e5、Nomic Embed、Stella、GTE-Qwen2、Jina v3/v4/v5、NV-Embed-v2。

## 维度与 Matryoshka

- 维度越高质量略升但存算增大；**768–1024 维是 2026 实用甜点**。
- **Matryoshka (套娃, MRL)**：训练时信息前置，前若干维即可自足，**截断**可省存储而几乎不掉质量（OpenAI 3-large 截到 256 维仍胜旧 1536 维 ada-002）。OpenAI/Cohere/Voyage/Nomic 已标配。

## API vs 自托管

- **API**：零运维、顶级质量，但按 token 计费、**数据离开本地**（专有技术 KB 要慎）。
- **开源（BGE-M3/Qwen3）**：可自托管、推理免费、无数据外流、中文强——**本地中文技术 KB 首选**。配 [[重排rerank|BGE-reranker/Qwen3-Reranker]]。

## 与 llmwiki

llmwiki 现用 LanceDB 向量。若换/补嵌入模型，用 BGE-M3 可同时拿到 dense 向量和 sparse 字面信号，直接支撑 [[混合检索与RRF|hybrid]]，省一个 BM25 引擎。
