# HyDE（假设性文档嵌入）

> 「问题和文档长得不一样，怎么匹配？」先让模型**写一个假设性答案**，用答案去检索。论文 arXiv 2212.10496。属 [[查询改写]] 家族。

## 核心思想

问题（疑问句）和目标文档（陈述/解释）的措辞差异大，直接拿问题向量去找文档常匹配不上。HyDE：
1. LLM 针对 query 生成一段**假设性文档/答案**（可能含错，没关系）；
2. 嵌入这段假设文档；
3. 用它的向量去 [[向量检索|检索]] 真实文档。

因为「答案更像目标文档」，能修正问-档**词汇不匹配**，对 dense 检索提升明显，**对非英语也有帮助**（已实证）。

## 优劣

- **强**：零样本场景、问答措辞差异大时显著提召回；多语友好。
- **弱**：多一次 LLM 生成（延迟/成本）；假设文档跑偏会引入噪声。

## 与 llmwiki

中文技术 KB 里术语/问法差异大，HyDE 是大库正规军阶段（见 [[RAG增强llmwiki方案]]）值得加的查询侧增强。小库直接漫游通常不需要。
