# 工艺纪律（foundation/taste.md）

> 与 `aesthetics.md` 分工：aesthetics 管**美的原理**（对比、层级、留白……违反 = 不好看），本文件管**工艺纪律**（拨盘、黑名单、验收流程……违反 = 一眼 AI 味 / 平庸）。两者叠加生效，任何设计师人格都不得违反。
> 思想来源：taste-skill（github.com/Leonxlnx/taste-skill）反平庸方法论，按本仓设计师人格体系重铸。
> 修改权限：**仅人类**。循环执行者只能在 journal 中提案。
> 版本：v1.1（2026-07-06，人类指令下增补：二代 AI 痕迹清单 / T11 语言收敛·结构发散 / 暗色与光标纪律）
> 引用方式：人格文件与产出说明中标 `[T1]`–`[T11]` 即可。

平庸不是运气差，是没有流程。本文件把「不平庸」变成三段式流程：**动手前**（T1 拨盘 + T2 判读）→ **动手中**（T4–T11 纪律）→ **交付前**（Pre-Flight 机械验收）。

## T1 · 三拨盘（Dials）—— 风格是三个数字，不是一种感觉

每次产出前设定三个 1–10 拨盘，并在设计判读里声明：

- **变化度 V（Variance）**：布局不对称程度。1 = 全部居中对称的安全网格，10 = 杂志式实验排版。
- **动效度 M（Motion）**：动画复杂度。1–3 = 仅 hover/active；4–7 = 过渡 + 入场级联（transform/opacity）；8–10 = 滚动驱动编排（钉住、视差、滚动揭示）。
- **密度 D（Density）**：信息拥挤度。1 = 大留白一屏一句话，10 = 驾驶舱式高密仪表。

规则：
1. 每位设计师人格的 frontmatter 带默认拨盘 `dials`；可按本次内容气质各 ±1 微调，超出即换人（说明选错设计师了）。
2. 意图推断参考：政务/信任优先 → 3-4 / 2-3 / 4-5；高端消费 → 7-8 / 5-7 / 3-4；玩趣/实验 → 9-10 / 8-10 / 3-4；判断不了 → 用所选设计师默认值。
3. M ≤ 3 时视同用户开启了「减少动效」：不做任何自动播放的动画。

## T2 · 简报推理先行（Design Read）—— 先声明，再动手

写任何代码/分页之前，输出一行设计判读：

> 「我把它读作：为 **[受众]** 做的 **[类型：落地页/汇报/课件/…]**，气质 **[2–3 个氛围词]**，请 **[设计师名]**（拨盘 V/M/D = x/x/x）。」

这一行是全部后续决策的锚：人可以在第一行就纠偏，而不是在成品出来后返工。

## T3 · Pre-Flight 机械验收 —— 打不了勾就是没做完

交付前跑一遍文末的 Pre-Flight 清单。清单里每条都是**可数、可判定**的（数眉标、数格子、量行数），不存在「感觉还行」。任何一条打不了勾，返工后再交付——单条违反即整体不合格。

## T4 · AI 痕迹黑名单（AI-Tells）—— 一眼 AI 味的东西全部封死

以下元素除非用户明确要求，一律禁止：

- **占位人名/头像**：张三李四、John Doe、蛋形 SVG 头像。人名要有真实感，头像用真实感图片或干脆不放。
- **假精确数字**：99.99%、成立于 2020 服务 1234567 人。数字要么来自真实数据，要么明确标注为示意。
- **填充动词/空话**：赋能、引领、一站式、极致体验、Elevate、Seamless、Unleash。只用具体动词。
- **产品痕迹装饰**：Hero 里的版本标签（v0.6 / BETA）、滚动提示（↓ scroll）、章节编号眉标（00 / INDEX）、图片上叠的假摄影署名、营销页页脚的 Build 号。
- **div 拼的假产品截图**：假任务列表、假终端窗口、假仪表盘。要么用真图，要么不放。
- **默认 AI 紫**：紫色辉光渐变不许当默认值——除非人格色板本来就是它（如 keynote-tech 极光板），且要能说出理由。

**二代痕迹**（模型逃逸一代规则后长出的新模板脸，同样封死；本清单随发现持续更新）：

- **图标芯片矩阵**：每张特性卡 = 线性图标 + 浅色圆角方块底 + 标题上方堆叠，成排复制。一个可以，一排就是签名。
- **侧边彩色竖条卡**：`border-left` 粗色条卡片——被多方点名为最明显的 AI 生成 UI 标志。
- **✨徽章三件套**：眉标 pill + 巨型全居中标题 + 双 CTA 的 stock hero 组合。
- **幽灵序号装饰**：01 / 02 / 03 大号淡色数字当段落装饰（章节编号眉标的二代变体）。
- **斜体衬线 display 标题**：已成“通用 AI 创业落地页”的新签名；同理警惕“米色底 + 衬线”当默认体面感（过度矫正也是味）。
- **Tailwind 默认脸**：`#4F46E5`/indigo-500 一族强调色、全站 16px 统一圆角 + 24px 统一 padding、无理由的 Inter/Space Grotesk。
- **渐变指标数字**：渐变文字只许落大标题关键词，落在数据/指标/正文上即装饰压过语义。
- **emoji 当图标**：一行五个失控色相，瞬间击穿单强调色纪律。
- **彩虹状态**：每行不同颜色徽章、“正常”状态也上色。状态色映射严重程度，正常态必须是灰。

## T5 · 布局力学（Layout Mechanics）—— 首屏与节奏的硬规则

- **首屏纪律**（web Hero / deck 封面）：标题 ≤ 2 行、副文 ≤ 20 词且 ≤ 4 行、行动点（CTA/结论）不滚动可见；文字堆叠 ≤ 4 个元素（眉标、标题、副文、CTA）。塞不下删内容，不缩字号 [A3]。
- **Logo 墙 / 信任条**必须在首屏之外单独成节，绝不塞进 Hero。
- **版式家族不重复**：8 节内容 ≥ 4 种版式家族（左右分栏 / 便当格 / 全幅金句 / 纵向叙事…）。同款左右交替（zigzag）≤ 2 连，第 3 个即返工 [A7]。
- **便当格铁律**：格数 = 内容数（3 件事 3 格，绝不为凑格拆内容或留空格）；≥ 2–3 格要有真实视觉差异（图 / 渐变 / 纹理），不许全白格拼盘。
- **禁偷懒分屏**：「左大标题 + 右小字解释」不许当默认——要么纵排（标题上、≤65ch 正文下），要么右栏放真正的视觉/交互物。
- **眉标节制**：眉标（uppercase 小标签）≤ ⌈节数/3⌉。上一节用了，接下来两节不许用。

## T6 · 动效纪律（Motion Discipline）—— 每个动效都要能说出理由

- 只动 `transform` 与 `opacity`；绝不动 top/left/width/height [性能铁律]。
- 绝不 `window.addEventListener('scroll')`——用 IntersectionObserver、CSS 滚动驱动或成熟库。
- 绝不默认缓动（linear / ease-in-out）——用自定义曲线，如 `cubic-bezier(0.16,1,0.3,1)`（果断收尾）或 `cubic-bezier(0.32,0.72,0,1)`（高级顺滑）。
- 每个动效要能用一句话回答「它向观者传达了什么」（层级 / 反馈 / 状态切换 / 叙事）。答不出的删掉 [A10]。
- 跑马灯（marquee）每页 ≤ 1 个。第 2 个 = 偷懒填充。
- `prefers-reduced-motion` 必须收敛到静态终值，终值必须独立成立可读 [A1]。
- 模糊（backdrop-filter）与噪点滤镜只放 fixed/sticky 元素或固定伪元素，绝不放滚动容器（手机掉帧死刑）。且全页 backdrop-blur 限导航 / 模态 / 至多 1 张特性卡——真机实测整页玻璃 FPS 掉 15–30%。
- 绝不弹簧超调（bounce/overshoot）缓动——已过时且廉价；果断收尾或高级顺滑二选一。
- 图片 hover 禁 scale/rotate（已是“生成 UI”特征）；交互反馈放在文字、边框、遮罩上。
- 自定义光标（跟随体/放大圈）只在桌面 `pointer:fine` 启用，触屏禁；跟随用 transform + lerp（系数 0.1–0.2），绝不高频重绘。
- 微交互投入按 ROI 排序：CTA 按钮与表单反馈 > 入场动效 > 装饰动效。

## T7 · 色板锁与禁色轮换（Palette Lock）—— 一个强调色，用到底

- **一板一色**：整个产出一个强调色，全页贯穿——暖灰页面不许第 7 节突然冒出蓝色 CTA [A5]。
- 强调色饱和度 < 80%；大面积色一律降饱和/调明度。
- **禁用被 AI 用烂的「高端感」默认色板**：米色 + 黄铜 + 酒红 + 咖啡这一族（#f5f1ea/#b08947/#9a2436 一类组合）。高端感请轮换：冷奢银铬、森林绿 + 骨白、黑 + 驼、钴蓝 + 奶油、赤陶 + 板岩。
- 连续同类项目不许复用同一色板（人格的「拿手三套系」正是为轮换而设）。
- 暗色板补充律 [A9]：背景禁纯黑 #000（用 L 8–12% 深灰）；前景彩色降饱和（蓝/青 −20~30%，红/橙 −10~15%）；深底禁**装饰性**彩色辉光——辉光必须是焦点信息自己在发光，不指向信息的 glow 一律删。
- 色 token 语义化命名：`--accent-action` 而非 `--gradient-start`——每个颜色要能回答“它表示什么”。

## T8 · 文案自审（Copy Self-Audit）—— 交付前扫一遍每一句话

- 语法坏句、指代不明的句子：改成朴素功能语言。
- 强行比喻、可爱但不对的文字游戏（AI 幻觉腔）：删。
- AI 文案指纹：破折号连用、超级动词（supercharge/empower/引领/极致，T4 已禁）、每节结尾“短句反转”的格言节奏、对冲语言（“或许能帮助”）——全部改成创始人声音：具体动词 + 具体名词，说产品实际做什么。
- 一页一种文案语域：技术等宽腔 / 编辑叙事腔 / 营销冲击腔，不混用（除非品牌声音明确要求）。
- 引用/金句 ≤ 3 行，署名干净（姓名 + 头衔即可），不编造。

## T9 · 重设计协议（Redesign Protocol）—— 改别人的东西先审计

接到「改版 / 美化 / 重新设计」类任务，第一步定模式：

1. **新建**（没有旧物或用户同意推翻）→ 按正常流程走。
2. **保品牌演进**（现代化但不能不认识）→ 先审计：品牌 token（色 / 字 / logo / 圆角）、信息架构（导航树、转化路径）、值得保留的招牌交互、该退休的 AI 痕迹。改造杠杆按优先级：字体刷新 → 间距节奏 → 色彩校准 → 动效层 → 关键节重排 → 整块替换（最后手段）。
3. **推翻重来**（新视觉语言、旧内容）→ 视觉当新建做，内容与信息架构保留。

**永不静默更改**：URL/锚点、主导航文案、表单字段名与顺序、品牌 logo、法务/授权文案。改这些必须先说。

## T10 · 状态完备性（State Completeness）—— 页面不止一种状态（web 特供）

- 加载态：形状匹配最终布局的骨架屏，不是通用转圈。
- 空态：认真排版，并指明如何填充。
- 错误态：表单就地内联提示；瞬时消息才用 toast。
- 按钮 / 表单 / 焦点环全部过 WCAG AA 对比（白底白字按钮 = 一票否决）[A1]。

## T11 · 语言收敛，结构发散（Coherence）—— “像 AI”的最大来源是系统级单调

研究结论：观者判断“这是 AI 做的”，最强预测因子不是某个组件丑，而是**整站结构单调**。解法是一句话：**设计语言全站唯一，版面结构刻意发散**。

- **语言收敛**（这些每样只许一套）：圆角档位、投影配方、图标风格、字族组合、缓动曲线、强调色。散了就是风格腐烂 [A7/A11]。
- **结构发散**（这些必须有变化）：区块版式家族、卡片尺寸与跨度、对齐轴、疏密节奏。「全页垂直堆叠 + 全宽分区 + 内容全居中 + 每区一个眉标」= 教科书级 AI 味，直接返工。
- 卡片只给可交互或确需围合的对象；**禁卡中卡嵌套**——分组优先用留白、邻近性、分隔线 [A3]。
- 双字体系统：一个有个性的 display 字体 + 一个克制的正文字体（2 字族上限内做出对比）；全站单字体无对比即 AI 感。
- 真实资产优先：真产品截图/实拍 > 成体系的定制插画 > 库存照片；没有真图就删掉图位，不用 div 拼假截图顶替 [T4]。
- 对自己施压：交付前自问一次「这页哪里还 generic？怎样更有辨识度？」并实际改一处——写进流程，不靠一次生成。

## 遥测锚点（软触发可验证）

每份产物（HTML 页面 / 网页 deck）**首行**写一条注释，deck 的 pptx spec 则写进 spec 顶部注释字段：

```html
<!-- designer: <id> · dials <V>/<M>/<D> · preflight <通过数>/<总数> -->
```

它让「模型有没有真的选设计师、有没有真的跑验收」变成事后可查的一行字，也为 `rubric.md` 评分积累证据。
