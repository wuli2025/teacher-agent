# RAPTOR（递归摘要树检索）

> 「既要细节，又要全局。」把文档聚类+摘要成多层树，检索时可在不同抽象层取材。论文 arXiv 2401.18059 (2024)。

## 核心思想

1. 把切块**聚类**；
2. 每个簇用 LLM **生成摘要**；
3. 对摘要再聚类、再摘要……**递归向上**，建一棵从「原文细节」到「高层主题」的树；
4. 检索时跨层检索（叶子=细节，高层节点=全局概括），既能答细节问也能答「整体讲什么」。

## 优劣

- **强**：长文档、需要不同粒度信息、全局归纳问题；比单层切块更能覆盖跨段落主题。
- **弱**：建树要多次 LLM 摘要（索引成本高）；更新需重建相关子树。
- 与 [[GraphRAG]] 同样面向「全局 sensemaking」，但用**树（摘要层次）**而非**图（实体关系）**。

## 与 llmwiki

llmwiki 的 **MOC / index.md / 概念页**本质就是人/模型手写的「摘要层」。RAPTOR 给的启示：可让模型**自动为知识簇生成多层摘要页**（比如给每个目录、每个主题群生成一页「概览」），让漫游既能下钻细节又能上浮全局。与 [[知识补全与缺口检测]]、[[STORM与Co-STORM]] 配合可半自动维护这些摘要层。
